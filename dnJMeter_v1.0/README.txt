dnJMeter_v1.0为阉割版本：
原计划Linux模块亦远程以no-gui方式执行JMeter，
后因远程调用Linux执行时读取环境变量异常问题阉割需求；
待后续解决该问题后补充更新。

===============================================================

dnJMeter脚本共分三个模块：
1）windows系统no-gui界面执行JMeter功能
2）获取Linux系统no-gui界面执行JMeter的对应执行语句（非远程使用Linux系统执行JMeter）
3）多服务器SFTP上传打包压缩后的JMeter脚本等文件（仅上传，请自行解压）

===============================================================

发压模式说明：（windows系统或Linux系统均可，但Linux系统仅获取执行命令）
(1) local_no_report : 本地控制机发压，不生成测试报告；
(2) remote_assign_no_report : 指定远程压力机发压，不生成测试报告；
(3) remote_all_no_report : 所有远程压力机发压，不生成测试报告；
(4) local_get_report : 本地控制机发压，生成测试报告；
(5) remote_assign_get_report : 指定远程压力机发压，生成测试报告；
(6) remote_all_get_report : 所有远程压力机发压，生成测试报告；

Tips : 
1) 如选择了不生成测试报告的方式发压，后续如需获取对应结果文件的测试报告，则在控制台输入如下命令：
jmeter -g [name.jtl/name.csv] -o [destDir]
2) 如选择了远程压力机发压，请确保参数化文件均已上传至压力机对应路径！！！
3) 无论使用上述哪一模块，均需先行配置config.ini文件（各配置参数均为中文，不作详细描述）；
4) config.ini配置文件中三大模块下首项参数均为开关（值为1即开启，为0或空即关闭，为其他值时均默认为关闭，且只可其中一项为1，另外2项为0或空，否则报错）
